# ADR-0036: Full live validation — 100% functional coverage on real evidence, both runtimes

- **Status:** Accepted
- **Date:** 2026-07-17
- **Deciders:** @realkarych
- **Amends:** [ADR-0031](0031-multi-runtime-ingestion-codex.md) (the Codex live leg was scoped in stage 3 as a single bare-prompt advisory basket whose verdict is logged, not asserted; this ADR expands it to full MCP + subagent + skill-substitute parity and flips the reliable verdicts to asserted). Also promotes the live-gate **scope** — previously governed only by the PV-6b calibration review and the e2e production-scenarios design, neither an ADR — into a decision record.
- **Related:** [ADR-0027](0027-verification-layer-and-reliability-metrics.md) (the A-vs-A + seeded-regression calibration methodology this extends to the whole CLI surface), [ADR-0030](0030-interactive-session-import.md) (the `import` entry point now exercised on real evidence), [ADR-0034](0034-gate-self-check.md) (`calibrate` now exercised live), [ADR-0035](0035-wilcoxon-signed-rank.md) (the paired axis now exercised live); the [PV-6b live calibration review](../internal/reviews/2026-07-08-pv6b-live-calibration.md) and the [e2e production-scenarios design](../internal/superpowers/specs/2026-07-15-e2e-production-scenarios-design.md); basket / evidence / CLI compatibility surfaces ([VERSIONING.md](../VERSIONING.md)).

## Context

The paid live gate (`e2e/run.sh`, workflow `e2e-live.yml`, twice-weekly Mon+Thu) is the only place catacomb runs against **real** agent transcripts. Everything else — the hermetic prod scenarios, the Go unit suite — runs over fabricated evidence. That matters because the whole product claim is "an offline statistical gate that works on real coding-agent runs"; a contract only fabricated evidence ever exercises is a contract nobody has proven on the thing customers actually feed it.

The live gate today drives six heterogeneous `claude -p` baskets (presence, continuous, sql, subagent, skill, mcp) plus one optional `codex exec` basket, and asserts the calibration invariants (A-vs-A must not gate; each seeded regression must gate, attributed to the swapped instruction). A read of the full functional surface (`docs/internal/reviews`/blueprint, verified at `be58e2c`) found **~95 distinct functional items** (14 subcommands × their flags/modes) of which **only ~34 are exercised live**. The ~61 uncovered items include four **entire** standalone subcommands (`verify`, `calibrate`, `pack`, `import`), four of five `baseline` subcommands, most `regress`/`calibrate` threshold and format flags, `trends --json/--metric/--pareto`, `diff`'s asymmetric per-side scoping, `subgraph`'s range mode, and several `bench` lifecycle flags.

The decisive finding: **none of the 61 need a live model call.** `bench` is the only command that spends money; every uncovered item is an offline transform over evidence `bench` already wrote to disk. They can be asserted **after** the existing bench steps at zero incremental spend. So "100% live functional coverage" for Claude is essentially free — it was simply never wired.

Codex is the asymmetric case. ADR-0031 stage 3 shipped a `runtime: codex` bench spawn path and a single live basket (`basket-codex.yaml`) that answers a bare prompt — it never drives a real MCP server, a real subagent spawn, or a skill. The ingest side already handles all three (MCP `mcp_tool_call_*` events, child rollouts via `parent_thread_id`), proven hermetically (scenario 56). The gap is live basket authoring, not ingest capability — with one hard exception documented below (Codex skills).

## Decision

Expand the live gate to validate **100% of catacomb's functional surface on real evidence for both runtimes** (Claude Code and Codex), keeping the existing twice-weekly Mon+Thu schedule and the skip-clean posture for the paid Codex leg. Concretely:

1. **Claude 100% offline-transform coverage.** Append the ~61 uncovered items to `e2e/run.sh` as assertions **after** the relevant basket benches, reusing each basket's runs-dir. Zero incremental model spend, with one principled exception (`--error-delta`, see Limitations).

2. **Codex live parity, three modalities.** New paid baskets on `gpt-5.4-mini` (the cheapest *usable* Codex rung — nano is API-only, unavailable in Codex CLI): an MCP basket (per-invocation `-c mcp_servers.*` registration of the existing `mcp-e2ekit` server), a subagent basket (prompt-forced `spawn_agent` delegation, `-c agents.max_threads=2`), and a skill basket that verifies the **work product only** (see decision 4). Plus Codex offline-transform coverage mirroring the Claude additions over the Codex evidence these baskets produce.

3. **Mixed-model policy for Claude — deliberately NOT a blanket Haiku switch (user decision, load-bearing).** Haiku runs the `continuous` basket and every genuinely-new $0/near-$0 offline-coverage cell; **Sonnet stays pinned (via the existing per-task `CHILD_MODEL` env override) on the five delegation-sensitive baskets**: presence's checkpoint-mark task, sql, subagent, skill, mcp. This is evidence-gated, not taste: the repo already measured Haiku failing exactly this obedience class and pinned back to Sonnet — mark obedience 4/5 then 2/5, sql multi-step tool adherence ~60-80%, both below the ≥4/5 baseline-reliability floor the gate hard-asserts. A blanket swap would break the gate's own trustworthiness checks before the seeded-regression assertions are reached. `CHILD_MODEL` is already per-task, so the policy is enforced selectively and guarded by a preflight assertion that the five stay Sonnet.

4. **Codex skills = artifact-verify substitute, not a skill node (user decision, load-bearing).** Codex invokes a `SKILL.md` via an ordinary file-read tool call plus system-prompt injection — there is **no distinguishable skill-invocation event, tool name, or rollout record** (confirmed by two independent upstream sources and the repo's own ingest: `reduce/skill.go` only matches Claude's `Skill`/`SlashCommand` tool names). A structural skill-node regression is therefore **impossible to seed on Codex**. The Codex skill basket verifies the **artifact** (`verify_emit.py` on `out/result.csv`), gating on `verifier.pass` 1→0, plus an optional **soft** grep of the rollout's `function_call`/`exec_command` arguments for a read of `.agents/skills/e2e-emit/SKILL.md` (proves the file was read — the closest observable proxy, never claimed as node presence). Documented as a Codex platform limitation.

5. **Verdict posture per Codex basket — reliable regressions asserted, unknown-base-rate signals logged.** Where a seeded Codex regression is deterministic once wired — the existing basket-codex continuous `tokens_out` growth (terse vs verbose), and the MCP-node drop — flip the verdict from *logged* to **asserted**. Where the base rate is genuinely unknown — subagent `spawn_agent` is a discretionary, prompt-triggered model choice that may fire 0/3 under low reasoning effort — keep it **logged/soft** until enough live reps accrue to calibrate the floor, exactly the posture basket-codex already takes for a new runtime. The hermetic mirrors of all three are asserted (deterministic fixtures); only the *live* subagent verdict stays advisory.

6. **Hermetic-mirror discipline is the $0 validation contract.** `e2e/run.sh` cannot be run without real API auth, so every new live assertion is mirrored by a hermetic prod scenario that runs the **same catacomb invocation + same assertion** over fabricated evidence. The mirror is what an implementer (who has no API key) runs to prove the assertion logic; the live leg only substitutes real evidence for the fabricated fixture. Where a mirror already exists (most offline transforms) the task reuses it; where it does not (e.g. `--format markdown`, `--iqr-factor`, `--coverage-floor`, `--audit-*`, asymmetric `diff`, the Codex MCP/subagent/skill nodes) the task **adds** it. No live assertion ships without a $0 mirror.

## Alternatives considered

- **Blanket Haiku swap of the whole Claude gate** to cut spend. Rejected — the user decision above; the repo has direct measured evidence Haiku fails the mark and sql obedience floors, which would make the gate's own calibration assertions flaky. Mixed-model is the honest floor.
- **Synthesize a Codex skill node from a heuristic** (treat a read of a `SKILL.md` path as a skill invocation). Rejected — it fabricates a cross-runtime step identity against no stable contract, the exact anti-pattern ADR-0031 forbids ("Normalizing Codex tool names… fabricates cross-runtime step-identity the gate must not promise"). Artifact + soft grep is the honest substitute.
- **Hard-assert the Codex subagent spawn on n=3 live.** Rejected for v1 — delegation is a discretionary function call gated by prompt wording, not a builtin always-available action; a 0/3 run is plausible under `model_reasoning_effort=low`. Log until calibrated; the documented lever if it under-fires is bumping just that cell to `medium`.
- **A dedicated new failing-tool basket to make `--error-delta` free.** A wrapper that exits before the model produces no transcript, so no `error_rate` signal exists to gate — `--error-delta` genuinely needs a real transcript with a failing tool call (a few cents of Haiku) or it stays a documented gap. Kept as a single, flagged, near-$0 exception rather than pretending it is free.
- **Cover the remaining items only hermetically.** Rejected — hermetic already covers most of them; the entire point of this ADR is that the *real-evidence* contract is what customers exercise and what has never been proven. Hermetic is the $0 validation substrate, not the destination.

## Consequences

- The live gate goes from ~34 to the full ~95 functional items on **real** Claude evidence, and Codex gains live MCP + subagent + skill-substitute coverage plus its own offline-transform surface. The single-vendor and "does it work on real runs" objections both get their real-evidence answer.
- **New model spend is small and bounded:** the Claude offline additions are $0 (they reuse existing evidence); the only new Claude spend is the optional near-$0 Haiku `--error-delta` cell (a few cents, or skipped as a documented gap). Codex adds three `gpt-5.4-mini` baskets at low reasoning effort — pennies, token-billed with no reported dollar cost, so they never enter the dollar total. Total incremental: cents, not dollars.
- **Maintenance surface grows:** more live baskets, more wrapper scripts, and one hermetic prod scenario per new modality (57-codex-mcp, 58-codex-subagent, 59-codex-skill) plus mirror extensions to existing scenarios. This is the deliberate cost of the coverage; the hermetic-mirror discipline keeps each addition CI-provable at $0.
- **Honest, documented limitations persist:** Codex skills can never produce a structural skill node (platform limitation, artifact-only); the Codex subagent live verdict is advisory until calibrated; `--error-delta` is the one item that is not strictly $0; and the mixed-model policy means the gate is a Haiku+Sonnet suite by design, not a uniform run.
- The Codex leg stays **optional and skip-clean** — a missing or unauthenticated `codex` CLI skips every Codex basket and leaves the overall exit unaffected, unchanged from ADR-0031. The Mon+Thu schedule and the `workflow_dispatch` manual trigger are unchanged; implementers never run the paid legs — they land only on the schedule or a manual dispatch.
- CLI/basket/evidence compatibility surfaces are untouched: this ADR adds no flags and no schema fields. It is purely additional coverage of existing contracts (plus new e2e baskets and hermetic scenarios).
