# SP4 Audit (Deterministic Half) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development
> (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use
> checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add per-cell outlier flags to the regress report and a deterministic prompt-pack
export (`catacomb pack`), per the SP4 spec.

**Architecture:** One PR wave on branch `feat/audit-deterministic`, four tasks: (1) exported
per-cell extraction in aggregate; (2) audit block in regress (thresholds, wiring, render,
CLI flags); (3) `catacomb pack` subcommand (stride sampling + bundle + manifest +
INSTRUCTIONS.md); (4) hermetic-E2E assertions and docs. Everything additive: `Record.V`
stays 1, store schema stays 5, no new deps, no RNG.

**Tech Stack:** Go stdlib only; exact types, formulas, defaults, and strings in
`docs/specs/2026-07-13-sp4-audit-design.md` — the spec is the source of truth; read it
first, use it verbatim.

### Task 2: Audit block in regress

**Interfaces (spec §1, verbatim):** `Thresholds` += `AuditIQRFactor float64` (default 3.0),
`AuditRelDelta float64` (default 0.5); `Input` += `BaselineCells, CandidateCells
[]aggregate.Cell`; `Report` += `Audit *Audit` (JSON `audit`, omitempty);
`CellFlag{RunID string; Task string (omitempty); Metric string; Value, Median, Band
float64}` (JSON run_id/task/metric/value/median/band), `Audit{Baseline, Candidate
[]CellFlag}` (both omitempty). Rule: per metric over duration_ms/cost_usd/tokens_in/
tokens_out/turns, flag iff `|v − m| > max(AuditRelDelta·|m|, AuditIQRFactor·IQR)` with
nearest-rank median/P25/P75 over the group's cell values; groups with < 3 cells never
flag; `Audit` nil when no cells provided or no flag fired. Flags sorted by RunID then the
fixed metric order. `Task` = cell's `task` label. Human render epilogue line per flag:
`audit: candidate run r07 (task sql) tokens_out 1932 vs group median 243 (band 121.5)` —
`%g` values; `(task …)` omitted when the label is absent. CLI: `--audit-iqr-factor`,
`--audit-rel-delta`, validated > 0 (operational error, exit 2), plumbed like the existing
threshold flags; the cmd layer passes `aggregate.Cells(baseGroup)`/`aggregate.Cells(candGroup)`
into `Input`. Compare computes audit AFTER findings; `overallVerdict` untouched.

- [ ] **Step 1:** Failing tests: calibration table (PV-6a pattern) — value exactly at the
  band does not flag, epsilon beyond does; IQR=0 identical group flags only past the
  rel-delta floor; 2-cell group never flags; both groups flagged independently; multi-metric
  cell yields one flag per exceeded metric in fixed order; verdict/exit unchanged with
  flags present (drive real `Compare`, assert `OverallVerdict` identical with and without
  cells); nil-cells dormancy (JSON has no `audit` key — assert raw JSON); render golden
  lines incl. task-less cell; `DefaultThresholds` extended values; flag validation exit 2 +
  messages; JSON round-trip.
- [ ] **Step 2:** FAIL. **Step 3:** Implement `regress/audit.go` + wiring. **Step 4:**
  `go test -race ./regress/ ./cmd/... -count=1` PASS; TestGatePower untouched; 100%; lint
  0. **Step 5:** Commit `feat(regress): per-cell outlier audit notes`.

### Task 3: `catacomb pack`

**Interfaces (spec §2, verbatim):** `catacomb pack <selector> --runs-dir <dir> --out <dir>
[--sample N] [--db <path>]`; selector via the existing `label:`/`name:` resolution
machinery (same as `regress --runs-dir`, incl. `name:` baseline lookup through `--db`).
Validation: `--runs-dir`/`--out` required, `--sample` > 0 (default 3), empty selection and
pre-existing out dir are operational errors (exit 2; refuse, never merge). Sampling: sort
selected runs by RunID, indices `floor(i·len/N)` for i=0..N-1, deduplicated. Bundle: copy
each sampled evidence dir verbatim to `<out>/<run_id>/` (walk + copy, preserve relative
paths; no symlink following — evidence dirs contain none, refuse if encountered).
Manifest `<out>/pack.json`: `PackManifest{Selector, RunsDir, SampleRule ("runid-stride"),
Requested int, Runs []string, CreatedAt time.Time}` (JSON selector/runs_dir/sample_rule/
requested/runs/created_at, indented like other JSON output). `<out>/INSTRUCTIONS.md`:
static template — bundle contents, what to inspect (shortcuts, gaming, tool misuse,
fabricated results), scores-JSONL return contract with a concrete example line
`{"key":"audit.clean","value":1,"run_id":"<run id>"}` and the follow-up
`regress --scores findings.jsonl --annotation audit.clean:higher-better` command.

- [ ] **Step 1:** Failing tests: stride table (N=1/N=3 over 5 and 15 runs — exact RunIDs
  pinned; N ≥ len → all); flag validation + missing-flag errors exit 2; existing out dir
  refused; empty selection refused; bundle completeness (fixture evidence dirs with
  subagents/artifacts/scores — every file byte-identical at the destination); manifest
  golden (all fields; `created_at` present — compare shape, not value); INSTRUCTIONS.md
  contains the example scores line and the regress command; `name:` selector path via a
  seeded test store.
- [ ] **Step 2:** FAIL. **Step 3:** Implement `cmd/catacomb/pack.go`, register in root.
  **Step 4:** `go test -race ./cmd/... -count=1` PASS; 100%; lint 0; gosec 0. **Step 5:**
  Commit `feat(cli): catacomb pack — deterministic prompt-pack export`.
