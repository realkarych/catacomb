# Milestone C — bubbletea TUI (`catacomb observe`) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship `catacomb observe [hash]` — an interactive bubbletea master-detail TUI that mirrors the web hero flow (sessions list → select a session → its execution graph as an expandable tree → select a node → node detail with tokens in/out, cost+provenance, duration, status, model, type, opt-in content) over the **same** daemon SSE feed and canonical graph model the web UI already consumes.

**Architecture.** The TUI is **another client of the existing daemon**, not a new backend. It reuses `GET /v1/sessions`, `GET /v1/sessions/{hash}/graph`, `GET /v1/subscribe?session=…` (with `Last-Event-ID`/`?since=` resume), and the opt-in `GET /v1/sessions/{hash}/nodes/{nodeId}/payload`, all bearer-gated via `daemon.ReadDiscovery`. A new `tui` package holds: (1) an **injected client seam** (`Client` interface the bubbletea models declare) with a real HTTP/SSE implementation extracted from the shared shape `cmd/catacomb/watch.go` already proves; (2) a **pure Go port of the web UI's presentation logic** (`format`, provenance, node-type/status vocab) so labels/wording match exactly; (3) a **pure graph→tree reducer** that applies the same delta envelope the SSE snapshot + tail emit (consuming the reduced graph the server already builds — it does NOT re-implement reconciliation); (4) the bubbletea `Model`s (root + sessions-list + tree + detail) whose `Update(msg)`/`View()` are pure and tested by direct calls. `cmd/catacomb/observe.go` is a thin cobra command that wires the real `Client` into `tui.NewModel` and runs `tea.NewProgram`. Everything outside the ~10-line `tea.NewProgram(...).Run()` call is pure and 100%-coverable.

**Tech Stack.** Go 1.26 stdlib (`net/http`, `net/url`, `bufio`, `context`, `encoding/json`, `errors`, `time`, `sort`/`slices`, `strings`, `fmt`, `io`); `github.com/spf13/cobra` (in tree); `github.com/stretchr/testify` (in tree). **New Go deps (this is the TUI milestone):** `github.com/charmbracelet/bubbletea v1.3.10`, `github.com/charmbracelet/lipgloss v1.1.0`, `github.com/charmbracelet/bubbles v1.0.0`. All pure Go, no cgo, cross-platform (linux/darwin/windows). The price table / pricing engine is **not** re-implemented here — cost arrives pre-computed in `Node.CostUSD` + `attrs["cost_source"]` from the daemon (Milestone A backend). Redaction is the daemon's job (Milestone B); the TUI only renders the `PayloadView` it returns.

## 100% coverage approach for bubbletea (the crux)

This is how every bubbletea file reaches 100% under `-race` with zero `time.Sleep` and zero real terminal:

1. **`Update(msg tea.Msg) (tea.Model, tea.Cmd)` and `View() string` are pure functions of model state + message.** Tests construct a model value, call `m.Update(someMsg)`, type-assert the returned `tea.Model` back to the concrete type, and assert its fields. Every keypress branch is exercised by feeding the corresponding `tea.KeyMsg` (e.g. `tea.KeyMsg{Type: tea.KeyRunes, Runes: []rune("j")}` or `tea.KeyMsg{Type: tea.KeyEnter}`). `View()` is asserted with `strings.Contains` on the rendered string for the labels/values that must appear. This is deterministic and needs no program loop.
2. **All I/O is a `tea.Cmd` returned by `Update`, and the `Client` behind it is an injected interface.** `Update` never blocks on I/O; it returns a `tea.Cmd` (a `func() tea.Msg`). Tests **invoke the returned cmd directly** (`msg := cmd()`) against a **fake `Client`** and assert the resulting `tea.Msg`, then feed that msg back into `Update`. The fake `Client` returns canned data or errors — no daemon, no HTTP, no goroutines, no clock.
3. **The streaming subscription is a channel the model drains via a `tea.Cmd` that does one non-blocking-ish read.** The `Client.Subscribe` returns a `<-chan StreamMsg` (events + a terminal error/close); the model's `waitForEvent` cmd does `m := <-ch; return m` and re-issues itself on each received event (the standard bubbletea channel pattern). Tests build a fake channel, push events, and drive the cmd directly — `require.Eventually`/buffered-channel discipline, never `time.Sleep`.
4. **`NO_COLOR` / non-TTY rendering is a pure style decision.** A `styles(noColor bool)` constructor returns a lipgloss style set (plain when `noColor` or not a TTY); tests cover both branches by calling it with both values. The model stores its style set, so `View()` output is deterministic per branch.
5. **The only line not unit-tested is `tea.NewProgram(...).Run()` inside `cmd/catacomb/observe.go`'s production wiring** — and that is covered the same way `ui.go`/`up.go` cover their `exec`/browser seams: the program constructor is a package `var teaRun = func(p *tea.Program) (tea.Model, error) { return p.Run() }` (or equivalent), swapped in tests to a fake that returns immediately, so `runObserve`'s wiring is 100% without spinning a real TTY. (`cmd/catacomb/main.go` is already coverage-excluded; `observe.go` is NOT and must hit 100% via the seam swap.)
6. **Optional teatest smoke (NOT the coverage vehicle).** `github.com/charmbracelet/x/exp/teatest` can drive a full program for one end-to-end "list → enter → detail" smoke, but it is **not** how we reach 100% (it is timing-sensitive). Prefer direct `Update`/`View`. **Decision: do not add teatest** — direct Update/View + cmd-invocation covers every branch deterministically and avoids a 7th dependency. (Revisit only if a branch proves unreachable by direct calls — that would be a refactor signal.)

### The injected client seam (`tui/client.go`) — the model declares this interface

```go
package tui

import "context"

type StreamMsg struct {
	Event SseEvent
	Err   error
	Done  bool
}

type Client interface {
	Sessions(ctx context.Context) ([]SessionSummary, error)
	Graph(ctx context.Context, hash string) ([]SseEvent, error)
	Subscribe(ctx context.Context, hash string, sinceRev uint64) (<-chan StreamMsg, error)
	Payload(ctx context.Context, hash, nodeID string) (PayloadView, error)
}

type HTTPClient struct {
	Addr  string
	Token string
	HTTP  *http.Client
}

func NewHTTPClient(disc daemon.Discovery) *HTTPClient
```

- `Sessions` → `GET /v1/sessions` (Bearer header) → decode `[]SessionSummary`.
- `Graph` → `GET /v1/sessions/{hash}/graph` → decode `[]SseEvent` (the REST snapshot the model `Apply`s first); 404 → a typed `ErrSessionNotFound`-equivalent.
- `Subscribe` → `GET /v1/subscribe?session={hash}` (+ `Last-Event-ID: {sinceRev}` when `sinceRev>0`), Bearer header; spawns one goroutine that scans `data:` lines (the `watch.go` shape), JSON-decodes each into an `SseEvent`, and sends `StreamMsg{Event:…}` on the returned channel; on ctx-cancel/EOF sends `StreamMsg{Done:true}`; on error `StreamMsg{Err:…}` then closes. The goroutine exits on ctx cancel (defined exit).
- `Payload` → `GET /v1/sessions/{hash}/nodes/{nodeId}/payload` → decode `PayloadView`; **403 → a typed `ErrContentDisabled`** (distinct from 401 `ErrDaemonRestarted`) so the detail pane shows "content viewing disabled by the daemon"; 404 → `ErrPayloadNotFound`-equivalent.
- 401 on any call → `ErrDaemonRestarted`; dial failure → `ErrDaemonUnreachable` (reuse `cmd/catacomb` sentinels by returning errors the command's `renderErr` recognizes — `tui` defines its own `ErrContentDisabled`/`ErrPayloadNotFound`/`ErrSessionNotFound`; the auth/dial mapping returns sentinels `cmd/catacomb` already owns via `errors.Is` on `os.Err...` is not applicable here, so `tui` re-exports `ErrDaemonUnreachable`/`ErrDaemonRestarted` as its own sentinels and the command maps them). **Pin:** `tui` owns the sentinels it needs (`ErrDaemonUnreachable`, `ErrDaemonRestarted`, `ErrContentDisabled`, `ErrSessionNotFound`, `ErrPayloadNotFound`); `cmd/catacomb/observe.go`'s error printer maps `tui.ErrDaemonUnreachable`→its own message etc. (the two packages each keep their own sentinel; the command translates). This avoids `cmd`↔`tui` import cycles.

```go
package tui

import tea "github.com/charmbracelet/bubbletea"

type focus int

const (
	focusList focus = iota
	focusTree
	focusDetail
)

type Model struct {
	client   Client
	hash     string
	focus    focus
	width    int
	height   int
	debug    bool
	styles   Styles
	loading  bool
	err      error
	sessions sessionsState
	tree     treeState
	detail   detailState
}

func (m Model) Init() tea.Cmd
func (m Model) Update(msg tea.Msg) (tea.Model, tea.Cmd)
func (m Model) View() string
```

- `Init` returns the initial load cmd: if `hash != ""`, load that session's graph + subscribe; else load the sessions list. (`Init`'s cmd is invoked directly in tests against the fake `Client`.)
- `Update` routes `tea.WindowSizeMsg` (store w/h), `tea.KeyMsg` (quit on `q`/`ctrl+c`; `tab` cycles focus; `d` toggles debug; else delegate to the focused pane's `update*`), and each app msg (`sessionsLoadedMsg`→fill list; `graphLoadedMsg`→seed tree graph + start subscribe; `streamEventMsg`→`Apply` + re-flatten + re-issue `waitForEvent`; `payloadLoadedMsg`→fill detail; `errMsg`→set `err`). Selecting a session (enter in list) returns a cmd loading that graph; selecting a node (enter in tree) sets `detail` + returns a payload-nothing cmd (content is opt-in via `c`).
- `View` composes `lipgloss.JoinHorizontal(list | tree | detail)` sized to `width`/`height`; renders loading/empty/error states (terse, no chrome). Non-TTY/`NO_COLOR` flows through `styles`.

### Pure reducer (`tui/reducer.go`) + tree (`tui/tree.go`)

```go
package tui

type Graph struct {
	Nodes      map[string]Node
	Edges      map[string]Edge
	established map[string]bool
	statusRev  map[string]uint64
	tombstones map[string]uint64
}

func EmptyGraph() Graph
func Apply(g *Graph, ev SseEvent)

type TreeRow struct {
	Node     Node
	Depth    int
	HasKids  bool
}

func BuildTree(g Graph) []TreeRow
func Flatten(g Graph, expanded map[string]bool) []TreeRow
```

- `Apply` mirrors `reducer.ts`: `node_upsert` rev-guarded against established upserts; `node_status` is a field-wise patch that never establishes identity and is LWW by `statusRev` (fixes the status-before-upsert clobber); `node_merge` re-points edges + tombstones old id; `edge_upsert`/`edge_delete` tombstone-LWW. Determinism: same deltas any order → same `Graph` (proven by a permutation test).
- `Flatten` walks `parent_child` from roots (session first), orders siblings by `sequence` edges then `TStart` then id, emits only expanded subtrees with `Depth`. No node dropped (orphans attach at root) — mirrors the web "no orphan" rule.

```go
package tui

import (
	"fmt"
	"strconv"
	"strings"
	"time"
)

func Duration(ms *int64) string {
	if ms == nil {
		return "—"
	}
	v := *ms
	switch {
	case v < 1000:
		return fmt.Sprintf("%dms", v)
	case v < 60000:
		return fmt.Sprintf("%.1fs", float64(v)/1000)
	case v < 3600000:
		return fmt.Sprintf("%dm %02ds", v/60000, (v%60000)/1000)
	default:
		return fmt.Sprintf("%dh %02dm", v/3600000, (v%3600000)/60000)
	}
}

func Tokens(n *int64) string {
	if n == nil {
		return "—"
	}
	v := *n
	switch {
	case v == 0:
		return "0"
	case v >= 10000:
		return fmt.Sprintf("%.1fk", float64(v)/1000)
	default:
		return withThousands(v)
	}
}

func Cost(usd *float64) string {
	if usd == nil {
		return "—"
	}
	v := *usd
	switch {
	case v == 0:
		return "$0.00"
	case v < 0.01:
		return fmt.Sprintf("$%.4f", v)
	default:
		return fmt.Sprintf("$%.2f", v)
	}
}

func ShortHash(h string, n int) string {
	if h == "" {
		return "—"
	}
	r := []rune(h)
	if n >= len(r) {
		return h
	}
	return string(r[:n])
}

func Date(iso string) string {
	if iso == "" {
		return "—"
	}
	t, err := time.Parse(time.RFC3339, iso)
	if err != nil {
		return "—"
	}
	return t.Local().Format("Jan 2 15:04")
}

func withThousands(v int64) string {
	s := strconv.FormatInt(v, 10)
	var b strings.Builder
	pre := len(s) % 3
	if pre > 0 {
		b.WriteString(s[:pre])
	}
	for i := pre; i < len(s); i += 3 {
		if b.Len() > 0 {
			b.WriteByte(',')
		}
		b.WriteString(s[i : i+3])
	}
	return b.String()
}
```

```go
func newObserveCmd() *cobra.Command {

		Use:   "observe [hash]",
		Short: "Interactive terminal observer for a Claude session",
		Args:  cobra.MaximumNArgs(1),

			if len(args) == 1 {
				hash = args[0]
			}
			deps := observeDeps{
				readDiscovery: daemon.ReadDiscovery,
				discoveryPath: daemon.DiscoveryPath(),

			}
			return err
		},

	cmd.GroupID = groupObserve
	return cmd
}
```

**Open questions / spec ambiguities (resolve before execution):**

1. **bubbles dependency scope.** The plan pins `bubbles v1.0.0` but the pane models above are hand-rolled (lists/viewports built directly from `Flatten` + lipgloss) rather than using `bubbles/list`/`bubbles/viewport`. Hand-rolling gives full deterministic 100% coverage (bubbles' internal branches are hard to fully cover via our tests). **Recommendation:** drop the `bubbles` dependency entirely (use only `bubbletea` + `lipgloss`) unless the executor wants `bubbles/textinput` for the `/` filter. If kept, only `bubbles/textinput` is justified; confirm. (This affects C0's `go get` and the dep list.)
2. **bubbletea v1 vs v2.** v1.3.10 is the latest **stable** tag; a v2 line exists in RC. The plan pins **v1** (mature, matches the repo's "simplest thing that works" + no-surprises stance). Confirm the team is OK staying on v1 (no v2 RC).
3. **`Subscribe` lifecycle vs `context.Background()` in cmds (C8).** Cmd constructors use `context.Background()` because bubbletea cmds outlive a single `Update`; the SSE goroutine's exit is then tied to process teardown, not a per-program ctx. **Recommendation:** thread a program-scoped `context.Context` (cancel on `tea.Quit`) into `NewModel` so `Subscribe`'s goroutine has a clean exit on quit. Decide whether to add `ctx` to `NewModel`'s signature (changes the pinned `NewModel(Client, string, bool)` → `NewModel(context.Context, Client, string, bool)`). Low-risk; flagged so the signature is settled before C12/C13.
4. **Content opt-in availability signal.** The daemon returns **403** when `allowPayloadAccess` is off (Milestone B). The TUI surfaces "content viewing disabled by the daemon" only after the user presses `c` and gets the 403 (lazy). There is no pre-flight capability probe. **Recommendation:** accept the lazy 403 (silence-when-healthy: no chrome advertising a disabled feature). Confirm vs. a startup capability check.
5. **Relative-time refresh (`tickMsg`).** Wall-time on running sessions/nodes drifts; a periodic `tea.Tick` would refresh the displayed durations. This adds a timer cmd (still test-coverable by invoking the tick handler directly). **Recommendation:** include a 1 s `tea.Tick` for live duration refresh (covered by feeding `tickMsg` to `Update`); it does not violate the no-`time.Sleep`-in-tests rule (tests feed the msg, never sleep). Confirm whether the minimalist bar wants a live-ticking clock or static-until-delta (the latter is quieter; I lean static-until-delta — durations update when a delta arrives, no background tick — to honor silence-when-healthy). **Leaning: NO background tick.**
