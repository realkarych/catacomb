# M1c-1 — CDC delta bus + in-Graph delta emission Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the pure-Go `cdc` package (a `GraphDelta` union with 8 kinds + a `Bus`/`Consumer` fan-out with coalesce-to-latest drop), make the reducer emit deltas from an in-`Graph` buffer drained per apply, wire the daemon to publish live deltas (and DISCARD reload/Recover deltas), and expose `deltas_dropped` in `/metrics` — all independently testable by a test consumer, with NO exporter and NO new dependencies.

**Architecture:** Settling spec §6.4's open decision in favor of **option (a), the in-`Graph` delta buffer**: `reduce.Graph` gains an unexported `deltas []cdc.GraphDelta` slice, an `emit(d)` helper, and a `DrainDeltas() []cdc.GraphDelta` method that returns and clears the buffer. Every mutation path in `reduce/reduce.go` emits coarse full-state deltas (`node_upsert`/`edge_upsert`/`run_started`/`session_ended`/`run_ended`/`node_status`) with `Rev` set to the triggering observation's `seq`; consumers are idempotent rev-guarded full-state upserts, so coarse emission is correct. The import graph is `model ← cdc ← reduce` and `model ← reduce` (no cycle). The `cdc.Bus` carries its OWN mutex (independent of the daemon's `d.mu`); `Publish` does only a non-blocking channel send plus the drop+coalesce bookkeeping, so it never blocks on consumer logic and is safe to call while holding `d.mu`. The daemon creates one `Bus` in `New` (lives for daemon life), drains+publishes after `Apply` in the LIVE path (`applyAndPersist`), and drains+DISCARDS in the reload (`ingestLocked`/`IngestOTLP` lazy reload) and `Recover` replay paths so historical deltas are never republished. `metricsSnapshot()` aggregates every consumer's `Dropped()` into a new `deltas_dropped` field.

**Tech Stack:** Go 1.26, pure-Go (no new deps; `modernc.org/sqlite` remains the only SQLite binding). `testify` (`assert`/`require`). White-box tests (`package cdc`, `package reduce`, `package daemon`). Concurrency tests use channels / `require.Eventually` — never `time.Sleep` (forbidigo).

## Key design decision 1: settling the §6.4 emission seam

**Decision: option (a), the in-`Graph` delta buffer.** Spec §6.4 leaves this open and recommends (a). The reducer already knows exactly what it mutated, so emitting at the mutation site is authoritative and O(1) per change; option (b) (before/after diff in the daemon) is O(N) per observation and duplicates knowledge the reducer already has. Concretely:

- `reduce.Graph` gains an unexported `deltas []cdc.GraphDelta` field (init `nil`; appended to lazily).
- `func (g *Graph) emit(d cdc.GraphDelta)` appends to `g.deltas`.
- `func (g *Graph) DrainDeltas() []cdc.GraphDelta` returns `g.deltas` and sets `g.deltas = nil` (returns AND clears).

This means `reduce` imports `cdc`, and `cdc` imports `model` (the deltas carry `*model.Node`/`*model.Edge`). `model` imports neither. **No import cycle:** `model ← cdc ← reduce`, `model ← reduce`. Confirmed by inspection — `model` has zero internal imports; `cdc` will import only `model`; `reduce` already imports `model` and will add `cdc`.

## Key design decision 2: coarse, rev-guarded full-state emission

Consumers are idempotent rev-guarded full-state upserts (ADR-0015 §1: apply a delta only if `delta.Rev > sink_rev`). That makes **coarse emission correct**: each `Apply` case emits a `node_upsert` carrying the FULL `*model.Node` for the primary node it touched, with `Rev = o.Seq`. We do NOT compute a minimal field-level diff. Emission rules:

- **`node_upsert`** (`Rev = o.Seq`, `Node` = the node's current pointer, `RunID`/`ExecutionID` from `o`): emitted by every case that calls `g.node(...)` + `g.stamp(...)` for its primary node — `session_start`, `session_end`, `user_prompt`, `assistant_turn`, `applyTool`, `applySubagent`, `marker`.
- **`edge_upsert`** (`Rev = seq`, `Edge` = the edge pointer): emitted by `upsertEdge` when it CREATES an edge OR re-touches an existing one (raises `e.Rev`). A re-touch with `seq <= e.Rev` does NOT emit (no state change).
- **`run_started`** (`RunID`, `ExecutionID`): emitted by `ensureRun` when it CREATES a new `*model.Run` (the not-`ok` map miss). A re-open from `abandoned` does NOT emit `run_started` (the run already exists in the consumer; its later lifecycle deltas convey the change). Re-opens are vanishingly rare and reload-only-correctness is preserved because reload discards anyway.
- **`session_ended`** (`RunID`, `ExecutionID`): emitted in the `session_end` case (the OTLP exporter in M1c-2 keys finalization on this).
- **`run_ended`** (`RunID`, `ExecutionID`): emitted in `applyRunEnded` when it actually abandons the run (not on the early-return `StatusOK` path).
- **`node_status`** (`Rev = triggering seq`, `Node` = the descendant's current pointer): emitted by the cascade (`applyCascade`) and by `closeIfOpen` for EACH node whose status actually changed — see Key design decision 3.

**Pointers vs. copies — published deltas carry a shallow COPY (race-safety for async consumers).** The whole point of the bus is async fan-out: a consumer goroutine (the M1c-2 exporter; later M3 streaming) reads `delta.Node` fields OUTSIDE `d.mu`, concurrently with live ingests mutating the same `*model.Node` UNDER `d.mu`. Reading the live pointer async is a DATA RACE (`-race` flags it regardless of eventual convergence). The reducer's `emit` may carry the live pointer internally (cheap; it is only ever touched under `d.mu`), but **the daemon shallow-copies each delta's `Node`/`Edge` at the publish boundary** (T4 Step 4) — `nc := *delta.Node; delta.Node = &nc` — done under `d.mu` where the node is NOT being concurrently mutated, so the published delta carries a stable snapshot of every SCALAR and reassigned-pointer field (`Status`, `Rev`, `Type`, `Name`, `RunID`, `TStart`/`TEnd`/`TokensIn`/`TokensOut`/`CostUSD` — the reducer always REPLACES these pointers, never mutates their pointee). The shallow copy still SHARES reference fields that are mutated in place (`Attrs` via cascade's `cancel_cause`, `Sources` via append) — **async consumers MUST NOT read `Node.Attrs`/`Node.Sources`/`Node.Payload` without synchronization.** The M1c-2 exporter reads only scalar/token/timing fields (§7.4), so the shallow copy is sufficient; deep-copying the reference fields is deferred (YAGNI) until a consumer needs them. This keeps the reducer simple (it emits pointers) and centralizes the copy at the one publish boundary. M1c-1's test consumer also benefits (gets stable snapshots), and a dedicated test asserts the snapshot semantics.

## Key design decision 3: the M1b M-1 fix (cascade `node_status` with triggering seq)

Per the **M1b M-1 note** (ledger line 74) and spec §6.1, `applyCascade` mutates a descendant's `Status`/`Attrs["cancel_cause"]` WITHOUT bumping `n.Rev` (consistent with the pre-existing `closeIfOpen`; not a reload bug). So the emitted `node_status` delta MUST carry the **TRIGGERING observation's `seq`**, not the descendant's stale `n.Rev` — otherwise a rev-guarded consumer would drop the status change (the descendant's `Rev` may already exceed its own stale value, or simply not reflect this cascade).

Mechanism: thread the triggering `seq` through the cascade. `cascadeStatus` gains a `seq uint64` parameter; `applyCascade` and `closeIfOpen` gain `seq uint64` too. All call sites have a triggering `seq`:

- `applyTool` (cancelled/superseded trigger) — `o.Seq`.
- `session_end` case — `o.Seq`.
- `applyRunEnded` — `o.Seq` (both the `closeIfOpen(session)` and `cascadeStatus(session, unknown)` calls).

A `node_status` delta is emitted for EACH descendant whose status ACTUALLY changed — both the cancelled/superseded arm (`applyCascade` when `resolveStatus` raised the status) AND the `StatusUnknown`/`closeIfOpen` arm (a close from `running`/`pending` → `unknown` is a status change a consumer must see). `Rev = seq` (the triggering observation), `Node` = the descendant's current pointer.

## Key design decision 4: `node_merge` / `edge_delete` have NO live producer

Per spec §6.1 these two kinds are for provisional→canonical id rewrite (`node_merge`) and re-parent edge tombstones (`edge_delete`). **Neither has a live M1 producer:** the reducer keys every node by its canonical id from creation (`ToolCallID`, `AssistantTurnID`, …) — there is no provisional id and no rewrite — and M1 never deletes an edge (`upsertEdge` only creates / raises `Rev`; there is no re-parent that removes an edge). The two enum constants are DEFINED (a constant referenced by no producer is not dead code — it is part of the wire contract M2/the exporter will use) but **NO emission code is written for them**. Writing unreachable `emit(DeltaNodeMerge…)` would be dead code and would break 100% genuine coverage. This is stated explicitly so a reviewer does not "complete" the set.

## Key design decision 5: coalesce-to-latest drop (the concurrency-tricky part)

Per spec §6.2, when a consumer's channel is full on `Publish` the bus must not block and must not permanently lose state; the guarantee is **eventual final state** — the consumer eventually receives every node's FINAL materialized state. Design (chosen for race-freedom with explicit seams, all under the bus's own mutex):

- Each `Consumer` holds: `ch chan GraphDelta` (buffered), `dropped int64`, and `dirty map[string]GraphDelta` (keyed by a coalescing key).
- **Coalescing key** = the delta's node ID for node-bearing kinds (`node_upsert`/`node_status` → `delta.Node.ID`; `edge_upsert` → `"edge:"+delta.Edge.ID`); lifecycle kinds (`run_started`/`session_ended`/`run_ended`) are keyed by `string(Kind)+":"+RunID` so a run's lifecycle close is never coalesced away by node churn and is itself idempotent. This guarantees "latest state per node/edge/lifecycle-event" semantics.
- **`Publish(d)`** holds the bus mutex, then for each consumer calls `c.deliver(d)`. `deliver` first opportunistically FLUSHES any `dirty` entries whose re-enqueue now fits (drain check: try a non-blocking send per dirty entry; on success delete it from `dirty`). Then it attempts a non-blocking send of `d`:
  - `select { case c.ch <- d: default: ... }`. On success, done.
  - On a full channel (`default`): store `c.dirty[key] = d` (overwriting any older entry for the same key with the LATEST), and `atomic`-free `c.dropped++` (we are under the bus mutex, so a plain increment is safe; `Dropped()` also reads under the bus mutex). This is the drop+coalesce path.
- **Re-emit** happens opportunistically at the NEXT `Publish` (the flush-dirty-first step) — when the consumer has read from `ch` and freed space, the next publish flushes the coalesced LATEST entries before enqueuing the new delta. This needs no background goroutine and no `time.Sleep`. Because `dirty` always holds the LATEST delta per key (overwrite-on-drop), and lifecycle entries are keyed so they cannot be coalesced away by node churn, the consumer eventually receives every key's final state once the channel has room — satisfying §6.2.
- **`Dropped() func() int64`** (the spec's `Consumer.Dropped` field is a func) returns a closure reading `c.dropped` under the bus mutex.
- **`Unsubscribe(c)`** removes `c` from the bus's consumer slice under the bus mutex and closes `c.ch` so a ranging consumer goroutine exits. Publishing after unsubscribe never touches `c` (it is gone from the slice).

This is fully testable with explicit seams: a test `Subscribe(0)` or `Subscribe(1)`, fills the channel, publishes more to force the drop+dirty path (assert `Dropped() > 0`), reads to free space, publishes again to force the flush/re-emit path (assert the coalesced LATEST delta arrives), and finally `Unsubscribe` (assert the channel closes). No sleeps — reads/sends are synchronous; cross-goroutine cases use `require.Eventually`.

```go
package cdc

import (
	"sync"

	"github.com/realkarych/catacomb/model"
)

type GraphDeltaKind string

const (
	DeltaNodeUpsert   GraphDeltaKind = "node_upsert"
	DeltaEdgeUpsert   GraphDeltaKind = "edge_upsert"
	DeltaNodeStatus   GraphDeltaKind = "node_status"
	DeltaNodeMerge    GraphDeltaKind = "node_merge"
	DeltaEdgeDelete   GraphDeltaKind = "edge_delete"
	DeltaRunStarted   GraphDeltaKind = "run_started"
	DeltaSessionEnded GraphDeltaKind = "session_ended"
	DeltaRunEnded     GraphDeltaKind = "run_ended"
)

type GraphDelta struct {
	Kind        GraphDeltaKind
	Rev         uint64
	Node        *model.Node
	Edge        *model.Edge
	OldID       string
	NewID       string
	RunID       string
	ExecutionID string
}

type Consumer struct {
	C       <-chan GraphDelta
	Dropped func() int64
	ch      chan GraphDelta
	dropped int64
	dirty   map[string]GraphDelta
}

type Bus struct {
	mu        sync.Mutex
	consumers []*Consumer
}

func NewBus() *Bus {
	return &Bus{}
}

func (b *Bus) Subscribe(bufSize int) *Consumer {
	ch := make(chan GraphDelta, bufSize)
	c := &Consumer{C: ch, ch: ch, dirty: map[string]GraphDelta{}}
	c.Dropped = func() int64 {
		b.mu.Lock()
		defer b.mu.Unlock()
		return c.dropped
	}
	b.mu.Lock()
	b.consumers = append(b.consumers, c)
	b.mu.Unlock()
	return c
}

func (b *Bus) Unsubscribe(c *Consumer) {
	b.mu.Lock()
	defer b.mu.Unlock()
	for i, x := range b.consumers {
		if x == c {
			b.consumers = append(b.consumers[:i], b.consumers[i+1:]...)
			close(c.ch)
			return
		}
	}
}

func (b *Bus) Publish(d GraphDelta) {
	b.mu.Lock()
	defer b.mu.Unlock()
	for _, c := range b.consumers {
		c.ch <- d
	}
}
```

```go
func coalesceKey(d GraphDelta) string {
	switch d.Kind {
	case DeltaNodeUpsert, DeltaNodeStatus:
		if d.Node != nil {
			return d.Node.ID
		}
		return string(d.Kind)
	case DeltaEdgeUpsert, DeltaEdgeDelete:
		if d.Edge != nil {
			return "edge:" + d.Edge.ID
		}
		return string(d.Kind)
	default:
		return string(d.Kind) + ":" + d.RunID
	}
}

func (b *Bus) Publish(d GraphDelta) {
	b.mu.Lock()
	defer b.mu.Unlock()
	for _, c := range b.consumers {
		c.deliver(d)
	}
}

func (c *Consumer) deliver(d GraphDelta) {
	for k, pending := range c.dirty {
		select {
		case c.ch <- pending:
			delete(c.dirty, k)
		default:
		}
	}
	select {
	case c.ch <- d:
	default:
		c.dirty[coalesceKey(d)] = d
		c.dropped++
	}
}

func (b *Bus) TotalDropped() int64 {
	b.mu.Lock()
	defer b.mu.Unlock()
	var total int64
	for _, c := range b.consumers {
		total += c.dropped
	}
	return total
}

func (g *Graph) upsertEdge(executionID, runID, src, dst string, seq uint64) {

		e.Rev = seq

	}

func (g *Graph) cascadeStatus(rootID string, status model.Status, seq uint64) {
	children := map[string][]string{}
	for _, e := range g.Edges {
		if e.Type == model.EdgeParentChild {
			children[e.Src] = append(children[e.Src], e.Dst)
		}
	}
	seen := map[string]bool{rootID: true}
	queue := []string{rootID}
	for len(queue) > 0 {
		cur := queue[0]
		queue = queue[1:]
		for _, c := range children[cur] {
			if seen[c] {
				continue
			}
			seen[c] = true
			queue = append(queue, c)
			g.applyCascade(c, rootID, status, seq)
		}
	}
}

func (g *Graph) applyCascade(id, rootID string, status model.Status, seq uint64) {
	if status == model.StatusUnknown {
		g.closeIfOpen(id, status, seq)
		return
	}
	n := g.Nodes[id]
	if n == nil || rank(n.Status) >= 3 {
		return
	}
	n.Status = resolveStatus(n.Status, status)
	if n.Attrs == nil {
		n.Attrs = map[string]any{}
	}
	n.Attrs["cancel_cause"] = rootID
	g.emit(cdc.GraphDelta{Kind: cdc.DeltaNodeStatus, Rev: seq, Node: n, RunID: n.RunID, ExecutionID: executionOf(n.ID)})
}

func (g *Graph) closeIfOpen(id string, status model.Status, seq uint64) {
	n := g.Nodes[id]
	if n == nil {
		return
	}
	if n.Status == model.StatusRunning || n.Status == model.StatusPending {
		n.Status = resolveStatus(n.Status, status)
		g.emit(cdc.GraphDelta{Kind: cdc.DeltaNodeStatus, Rev: seq, Node: n, RunID: n.RunID, ExecutionID: executionOf(n.ID)})
	}
}
```

**Critical correctness — reload/Recover discard.** `g.ApplyAll(prior)` (lazy reload) and the `Recover` replay loop accumulate deltas in the buffer. Those describe HISTORICAL state the consumer either already has or will get via a future snapshot (M1c-2 snapshot-then-attach). Republishing them would deliver stale/duplicate/out-of-order history. So after each reload `ApplyAll` and after the `Recover` loop, call `g.DrainDeltas()` and throw the result away — this also bounds the buffer so it never grows unboundedly across reloads. The live `applyAndPersist` is the ONLY place that publishes.

func (d *Daemon) publishDelta(delta cdc.GraphDelta) {

		nc := *delta.Node

	}

		ec := *delta.Edge
		delta.Edge = &ec
	}
	d.bus.Publish(delta)
}
