# M0.3 ŌĆö Graph finalization & daemon resilience (design)

- **Status:** Approved (brainstorm)
- **Date:** 2026-06-21
- **Deciders:** @realkarych
- **Builds on:** M0.2b (live hook-ingestion daemon, merged in PR #11)
- **Implements (subsets of):** ADR-0012 (node finalization & run lifecycle), ADR-0014 clause-2 (status lattice), ADR-0019 clause-1 (fault isolation), ADR-0005 (run model)
- **Related spec sections:** ┬¦5.2, ┬¦5.4, ┬¦5.8, ┬¦7, ┬¦8, ┬¦13, ┬¦16, ┬¦17

## 1. Context & goal

M0.2 shipped the live hook-ingestion path: a daemon receives Claude Code hook
events over loopback TCP + bearer token, reduces them into per-execution
graphs, persists to SQLite, and recovers on boot. The main design spec (┬¦18)
jumps from M0.2 straight to M1 (OTel). **M0.3 is the interstitial milestone
that makes the *live* graph trustworthy before new sources are added.**

Two concrete gaps undermine the realtime graph today:

1. **Nodes never finalize.** A `tool_call`/`subagent` whose terminal
   observation never arrives (crashed/OOM subagent, dropped MCP server,
   abandoned tool, a `PostToolUse` lost while the daemon was down) stays
   `running` forever; a run is never queryable as "complete." The status
   enum has no value for "outcome unknown," and the current lattice is
   wrong (see ┬¦3).
2. **One bad input can kill capture.** There is no fault-isolation
   boundary: an unrecovered panic on a malformed/unexpected hook payload
   crashes the daemon and loses every run's in-memory graph ŌĆö unacceptable
   for a process running alongside the operator's real sessions.

**Goal:** every node and run reaches a terminal status, runs are queryable as
complete/abandoned, and a single malformed hook never stops capture for other
runs ŌĆö all under the existing 100%-coverage, cross-platform, comment-free
discipline.

## 2. Scope

In scope (the "reliable live graph" cut):

- New statuses `unknown` (orphaned **node**) and `abandoned` (reaped **run**);
  a correct, order-independent **node status lattice**.
- **SessionEnd descendant-closure** ŌåÆ `unknown` for still-open descendants.
- A **runs table** + run lifecycle (open / ended / abandoned), as a
  materialized projection of the observation log.
- **Marker nodes** for PreCompact/Notification, with `trigger`/`message`
  enrichment (currently dropped).
- An **idle-reaper** that closes quiescent runs as `abandoned` via a durable
  synthetic `run_ended{reason:timeout}` observation.
- **Crash-isolation**: a `recover()` boundary at ingest + a **poison-quarantine
  table** retaining raw bytes for diagnosis.
- A one-time **live field-name verification** against a real Claude Code
  session (Step 7), run by the operator as the first task.

Out of scope ŌåÆ M0.4 / M1 (see ┬¦11).

## 3. Status model & lattice (`model` + `reduce`)

**Current state (to be fixed).** The enum is
`pending|running|ok|error|blocked|cancelled`. `strongerStatus(cur,next)`
returns `cur` only when `cur` is terminal and `next` is not, where terminal =
`{ok,error,blocked,cancelled}`. This treats `cancelled` as a hard terminal
with last-writer-wins among terminals ŌĆö **contradicting ADR-0014**, which
requires `cancelled`/`unknown` to be *provisional* (a genuine late `ok`/`error`
must supersede them). The current behavior is order-dependent and breaks the
┬¦16 commutativity invariant.

**Decision.**

- Add **`unknown`** (a **node** status: an orphaned/unclosed node we will not
  learn the outcome of) and **`abandoned`** (a **run** status: a reaped run ŌĆö
  ┬¦5/┬¦6). `unknown` participates in the node lattice below; `abandoned` is a
  run-lifecycle state, never applied to a node by this lattice. (`superseded`
  is deferred with the cascade that produces it ŌĆö ┬¦11.)
- Replace the node lattice with a rank function:
  - **genuine terminal** `{ok, error, blocked}` ŌåÆ rank 3 (latches; beats all),
  - **provisional terminal** `{cancelled, unknown}` ŌåÆ rank 2 (reversible by a
    later genuine terminal),
  - `running` ŌåÆ rank 1, `pending` ŌåÆ rank 0.
- `resolve(cur, next)`:
  - if `rank(cur) == 3` and `rank(next) < 3` ŌåÆ keep `cur` (genuine latches);
  - if `rank(next) == 3` ŌåÆ take `next` (genuine supersedes provisional/running);
  - else the higher rank wins; on a tie, take `next` (observations apply in
    `seq` order, so this is deterministic).

This makes a node's final status **independent of arrival order**: a genuine
terminal always wins regardless of when it arrives, and inferred statuses are
reversible. The rebuild invariant (replay in `seq` order ŌåÆ identical graph)
holds.

## 4. SessionEnd descendant-closure (`reduce`)

On a `session_end` observation, after stamping the session node:

- walk the `parent_child` closure rooted at the session node;
- every descendant whose status is non-terminal (`pending`/`running`) is set
  to **`unknown`** (provisional).

The closure is computed **inside the fold** (`Apply`), so a rebuild
reproduces it deterministically, and a later genuine `ok`/`error` for any
descendant supersedes the `unknown` via the lattice (┬¦3). Activity arriving
strictly *after* `session_end` (a pathological `seq` ordering) is not closed
here; the idle-reaper (┬¦6) is the backstop.

## 5. Runs table & lifecycle (`store` + `model`)

The observation log remains the system of record (ADR-0010). The **runs table
is a materialized projection of it**, rebuilt on `Recover()` from
`ObservationsSince(0)` ŌĆö so run state is always reconstructable and never the
authoritative copy.

- `model.Run` gains `Status Status` (`running`=open / `ok`=ended / `abandoned`),
  `EndReason string`, `LastSeq uint64`.
- New `runs` table keyed by `run_id`: `session_ids`, `status`, `started_at`,
  `ended_at`, `end_reason`, `last_seq`.
- Lifecycle: a run is **open** (`running`) on its first observation
  (`started_at` = first event time). It ends via:
  - `session_ended` on a **single-session run** (`run_id == session_id`, i.e.
    no explicit `CATACOMB_RUN_ID` grouping) ŌåÆ run `ok`. This is the common
    hooks-only case and avoids a cleanly-ended run sitting open until the
    reaper mislabels it `abandoned`. For a **grouped** run (explicit
    `CATACOMB_RUN_ID` spanning sessions), `session_ended` finalizes only that
    session, never the run (ADR-0012) ŌĆö the run ends via the reaper or a
    wrapper marker.
  - a synthetic `run_ended{reason:timeout}` (reaper, ┬¦6) ŌåÆ run `abandoned`;
  - (an explicit wrapper end-marker ŌåÆ run `ok` ŌĆö no producer in v1; the
    `end_reason` column exists for M2+).
- Run status is **reversible**: a new observation for an `abandoned` run
  (reawakening) returns it to `running`, mirroring the node lattice's
  provisional reversibility (ADR-0012 amendment).
- `Store` additions: `UpsertRun(model.Run) error`, `ListOpenRuns() ([]model.Run, error)`,
  `Runs() ([]model.Run, error)`.
- In v1 (hooks-only) `run_id Ōēł execution_id` (one session per execution; the
  daemon already shards graphs by `execution_id`). The table tracks `run_id`
  and rolls up its executions; multi-execution grouping via `CATACOMB_RUN_ID`
  is handled by the same row carrying multiple `session_ids`.

## 6. Idle-reaper (`daemon`, M0.3b)

A ticker goroutine bound to the daemon's `Serve` context (`time.Ticker` ŌĆö not
`time.Sleep`, which `forbidigo` bans). On each tick:

- read `ListOpenRuns()`; for each open run whose last observation is older
  than the **quiescence window** (default **30 min**, configurable via a
  `daemon` flag), the reaper **appends a synthetic observation**
  `run_ended{reason:timeout}` for that run.

The synthetic observation is durable and rebuildable (ADR-0010): on apply the
reducer sets the **run** `abandoned` and closes its **non-terminal descendant
nodes** (`pending`/`running`) to `unknown` (ADR-0012 clause-2; already-provisional
or genuine nodes are left as-is). "Now" comes through the established `nowFn`
seam.

- **Shard release:** after a cooldown Ōēź the window, the reaper drops the run's
  in-memory graph shard from the daemon (recoverable from the store).
- **Invariant:** `reaper-window < retention-TTL` (ADR-0012 amendment) ŌĆö a
  reaped run's observations are retained so a reawakened run can still have
  its inferred statuses superseded by a genuine terminal via rebuild.

The reaper is the owner/trigger for `run_ended`, the missing piece flagged in
ADR-0012.

## 7. Crash-isolation & poison-quarantine (`daemon` + `store`, M0.3b)

The hook receiver is request-per-event, so fault isolation is **per-request
`recover()`** at the `Ingest` boundary (no supervised-restart-with-backoff is
needed ŌĆö that is for the long-running adapters of M1/M2).

- A panic or a parse/reduce error during `Ingest` is converted into a
  **quarantine record** persisted to a new `quarantine` table ŌĆö `raw` bytes,
  `hook_type`, `err`, `at` ŌĆö plus an in-memory counter and a logged error.
  The daemon stays up and other runs keep ingesting.
- The forwarder remains **fail-open**: the HTTP handler still returns `204`
  (the hook already fired; the raw bytes are captured for diagnosis).
- `Store` addition: `Quarantine(rec model.QuarantineRecord) error`.
- **Invariant (with a fixture):** *one bad input never stops other runs* ŌĆö a
  poison payload interleaved with good ones leaves every good run's graph
  intact.

## 8. Marker nodes & enrichment (`ingest/hook` + `reduce`, M0.3a)

Today PreCompact/Notification parse to `kind:"marker"` with only
`{hook_event}` ŌĆö the already-parsed `trigger`/`message` are dropped, and the
reducer has **no `marker` case**, so markers produce no node.

- **Parser:** include `trigger` (PreCompact) and `message` (Notification) in
  the marker observation's attrs.
- **Reducer:** add a `marker` case creating a `NodeMarker` attached to the
  session via a `parent_child` edge, carrying `hook_event` + `trigger`/`message`
  in attrs. Markers are point-in-time (no status lifecycle).

## 9. Decomposition

Two plans, split along the layer seam (mirrors M0.2a/M0.2b):

- **M0.3a ŌĆö Graph finalization** (`model` + `reduce` + `store`): statuses &
  lattice (┬¦3), SessionEnd closure (┬¦4), runs table & lifecycle application
  (┬¦5), marker nodes (┬¦8). Pure, deterministic reduction ŌĆö tested **offline**
  via `ApplyAll`/replay, including **out-of-order `seq`** for commutativity.
  Step 7 (┬¦10) is task 0.
- **M0.3b ŌĆö Daemon liveness & resilience** (`daemon` + `store`): idle-reaper
  (┬¦6), crash-isolation & quarantine (┬¦7). The live/timer/robustness layer ŌĆö
  tested via the `nowFn` seam, an injectable ticker, and a panic-injection
  seam in the reduce path.

## 10. Step 7 ŌĆö live field-name verification (task 0 of M0.3a)

The reducer trusts `ingest/hook`'s envelope field names; M0.2's contract is
pinned only by committed fixtures. Before building the lattice on top, the
**operator** runs a short real Claude Code session against the daemon
(`catacomb daemon` + `install-hooks --project`, then a prompt + one tool call),
and we inspect the SQLite store together. If a persisted tool node lacks a
`name`/status, or `tool_use_id` is absent, the envelope tags differ from
reality ŌĆö we fix the tags **and** the nine fixtures together as the first
commit. (The agent cannot drive an interactive session and must not read the
operator's private transcripts or commit captured data.)

## 11. Out of scope (ŌåÆ M0.4 / M1)

- **Rest of ADR-0019:** health/metrics endpoint, `lossy`-run flagging,
  dogfooding loop-break, soft-caps.
- **Rest of ADR-0014:** `superseded` status, the transitive cancel/supersede
  cascade (clause-3), and the OTel-conditional structure precedence (clause-1)
  ŌĆö the latter needs an OTel source (M1).
- **Offline EOF closure** (JSONL EOF ŌåÆ `unknown`) ŌĆö needs the JSONL tailer (M2).
- **Token rotation.**

## 12. Constraints (inherited, binding)

- No comments in Go source except `//go:build`/`//go:embed`/`//go:generate`
  (enforced by `internal/codepolicy`).
- 100% line coverage under `-race`, TDD-first.
- Cross-platform (Linux/macOS/Windows): no `time.Sleep`, no unix-socket
  assumptions, no `0600`-mode assertions; time via `nowFn`.
- The observation log is the system of record; the runs table and all node
  statuses are derived and rebuildable from it.
