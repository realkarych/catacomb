# Internal development material

These are **historical, working documents** — implementation plans, design specs,
review notes, and agent-tooling roadmaps captured while catacomb was built. They are
kept for provenance and are **not part of the product or its documentation**.

Some describe designs that have since been **superseded** — most notably an earlier
real-time observability daemon that
[ADR-0026](../adr/0026-form-factor-pivot-offline-eval-gate.md) replaced with the current
offline eval gate. Treat anything here as a snapshot of intent at the time it was
written, not as a description of current behavior.

To *use* catacomb, start at the [documentation index](../README.md) and the
[user guide](../guide/README.md).
