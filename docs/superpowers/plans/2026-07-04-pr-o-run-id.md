# PR-O: Honor `CATACOMB_RUN_ID` (F9, ADR-0005) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Dispatch a fresh subagent per task with review between tasks; the orchestrating window only coordinates. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the exported-but-unconsumed `CATACOMB_RUN_ID` real (finding F9, `docs/reviews/2026-07-04-dogfood-calibration.md`). The forwarders send the run id on a new `X-Catacomb-Run-ID` header; the daemon overrides the parsers' `RunID: e.SessionID` default with it when present; multiple sessions sharing one run id are grouped into one logical run by the existing read-layer union. `catacomb runs`/`inspect` gain a `--run-id`/positional selector so bench manifest run ids become queryable in the store.

**Scope decision — (B) tagging-only, grouped-run lifecycle deliberately deferred.** The daemon holds **one graph per session** (`d.graphs[execID]`, `execID := execBySession[sessionID]`; `daemon/daemon.go:211-215,316-320`). Two sessions that share a run id therefore live in two separate graphs, each with its own `*model.Run` object keyed by the same id. The grouped view is computed **at read time** by `summarizeGraphs` (`daemon/sessions.go:208-319`), which folds every graph whose run matches: `Status` via `foldStatus` (`statusRank`: Error 3 > Running 2 > OK 1 > other 0), `StartedAt` = earliest, `EndedAt` = **latest**, cost/tokens/nodes summed. This means honoring the env var needs **no lifecycle change**: PR-P's `session_end`→run finalization (`reduce/reduce.go:40-56`, landed #116) stays exactly as-is and finalizes only its own session's per-graph run object; the union across sessions is derived on read. Option A (a shared run object finalized only on a wrapper-exit signal, with a reason-aware `applyRunEnded` and a `session_end` guard) would **partially revert #116** to build machinery for a shared object this architecture does not have — rejected. Roadmap item (4)'s wrapper run-end signal is not implemented; the idle reaper remains the fallback for sessions that die without a terminal event, unchanged.

**Multi-session status/`EndedAt` semantics encoded (and documented in Task 4):**

- Each session keeps its own per-graph run object; `session_end` finalizes only that session (PR-P unchanged, not reverted).
- Grouped **status** = highest-severity/liveness across sessions: **Error** if any session errored, else **Running** while any session is live, else **OK** (`foldStatus`). A run stays `running` in `runs`/`inspect` until its last live session ends.
- Grouped **`EndedAt`** = the **latest** session end (**"last session_end wins"**); `StartedAt` = earliest; `DurationMS` = the span. Grouped cost = Σ per-session reported-or-estimated total (PR-Q precedence applied per session, then summed — `daemon/sessions.go:258-293`, `aggregate/aggregate.go:356-383` already sum every `SessionTotal()` node, so a multi-session `RunGraph` sums each session's reported total correctly).
- **Live == Recover purity:** the run id rides the *observation* (`o.RunID`, persisted by `insertObservation`, `store/sqlite.go:177`), never daemon state. `Recover()` and the offline `storeGraphs` both replay observations through `Graph.ApplyAll`, so grouping reproduces identically after restart.

**Known Option-B boundaries (documented, not fixed — bench/regress are single-session so unaffected):**

- The persisted `runs` table row for a shared id is last-writer-wins across sessions, but **no read path reads it** for grouping (`runs`/`inspect`/`regress`/`bench` all reconstruct from observations via `storeGraphs`), so this is cosmetic.
- `aggregate`/`regress` `runDuration` reads the single anchor session's `RunGraph.Run` timing (`collectRuns` first-wins; `aggregate/aggregate.go:384`), not the union — nodes *are* unioned, only the Run-object timing is single-session. Bench cells are single-session, so every shipped `bench`/`regress` workflow is unaffected; `runs`/`inspect` (via `summarizeGraphs`) report the correct unioned span.

**Run-id entry mechanism per path:**

- **hook:** `catacomb hook` reads `os.Getenv("CATACOMB_RUN_ID")` (inherited: `catacomb run` → claude → hook process) and sets header `X-Catacomb-Run-ID`; `handleHook` reads the header into `IngestWithLabels`.
- **stream (`catacomb run`):** `runChildObserved` already exports `CATACOMB_RUN_ID` to the child and holds `runID` in scope; it passes `runID` to `streamForward`, which sets the header; `handleStreamJSON` reads it (constant per request) into `IngestStreamJSONWithLabels`.
- **stream (`catacomb ingest stream-json`):** reads `os.Getenv("CATACOMB_RUN_ID")` and sets the header.
- **bench:** `runBenchCell` already passes `cell.RunID` as the `runID` arg to `runChildObserved` (`cmd/catacomb/bench.go:156`), so the header flows with zero bench code change; `cell.RunID` (`bench-<basket>-<task>-<variant>-r<rep>`) already satisfies the run-id charset.
- **jsonl/otel:** untouched — transcripts and spans carry no env; those paths keep `RunID: e.SessionID`.

**Validation:** a run id is a single opaque token used as an HTTP header value, a graph run key, and a SQLite `runs` primary key. New `model.ValidRunID`: non-empty, ≤256 bytes, `^[A-Za-z0-9._-]+$` (a superset of bench `idCharset` and of UUID/ULID session ids, and header/DB-safe). Two gates: the forwarder only *sets* the header when `ValidRunID` (invalid never leaves the client), and the daemon `attachRunID` ignores an invalid header value (defense-in-depth for untrusted network input), **falling back** to the parser's `session_id` default — never quarantining, never erroring. `catacomb run --run-id` additionally validates at the CLI for a fast, friendly error.

package model

import "regexp"

const maxRunIDLen = 256

var runIDRe = regexp.MustCompile(`^[A-Za-z0-9._-]+$`)

func ValidRunID(s string) bool {

}

func (d *Daemon) IngestWithLabels(hookType string, payload []byte, labels, runID string) (err error) {
	d.mu.Lock()

	defer func() {

	return nil
}

		return fmt.Errorf("invalid --run-id %q: expected [A-Za-z0-9._-]{1,256}", runID)
	}
	s, err := openReadStore(open, dbPath)
	if err != nil {
		return err
	}
	defer func() { _ = s.Close() }()
	graphs, err := storeGraphs(s, mkPricer())
	if err != nil {
		return err
	}
	selector := model.ParseLabels(strings.Join(labels, ","))
	runs := collectRuns(graphs)
	summaries := make([]daemon.SessionSummary, 0, len(runs))
	for _, r := range runs {
		if runID != "" && r.ID != runID {
			continue
		}
		if !model.MatchLabels(r.Labels, selector) {
			continue
		}
		summaries = append(summaries, daemon.SummarizeRun(r.ID, graphs))
	}
	...
}
