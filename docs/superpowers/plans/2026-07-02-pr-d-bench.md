# PR-D: `catacomb bench` Basket Runner Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Execute a declarative basket (tasks × variants × k reps) through the existing `catacomb run` wrapper with runner-emitted task markers and a resumable manifest — the deterministic execution layer of ADR-0022 §3, completing the P0 roadmap.

**Architecture:** a pure `bench/` package owns the basket YAML schema, validation, deterministic cell expansion, basket content hashing, and manifest read/write; `cmd/catacomb/bench.go` executes cells sequentially by reusing the in-process `runChild` wrapper (generated run-id + conventional labels `basket`/`task`/`variant`/`rep`), observes the child's stream-json for the first `session_id` and posts `task:<id>` start/end markers via the existing mark client, records every cell in an incrementally-written JSONL manifest (resume skips completed cells), and prints a ready-made `catacomb regress` epilogue.

**Tech Stack:** Go 1.26, `gopkg.in/yaml.v3` (already a dependency via `config/`), stdlib, testify.

## Global Constraints

- No comments in Go code (only directives); enforced by `internal/codepolicy`.
- 100% test coverage (`make cover`); TDD — failing test first, RED evidence.
- `gofumpt`/`goimports`; `make lint` 0 (cache clean first if stale sibling issues).
- **Determinism:** cell expansion order is fixed (tasks in file order × variants in file order × rep 1..k); run-ids and labels are pure functions of (basket, task, variant, rep); the basket hash is a pure function of the file bytes. Manifest timestamps use a `nowFn` seam.
- Cell failure is RECORDED, never aborts the basket (unless `--fail-fast`); the bench exit code is 0 if all cells ran (even with failing cells — regression detection is `regress`'s job), 1 if `--fail-fast` tripped, 2 on operational errors (bad basket file, store/daemon unreachable at start, manifest IO errors).
- Sequential execution only (no parallel cells) — keeps daemon sharding and attribution simple (YAGNI until proven needed).
- No new daemon/store surface — client-side only.

### Task 1: `bench` package — basket schema, expansion, hash, manifest

**Files:**

- Create: `bench/basket.go`, `bench/manifest.go`
- Create: `bench/basket_test.go`, `bench/manifest_test.go`

**Interfaces (exact):**

```go
package bench

type Task struct {
	ID  string            `yaml:"id" json:"id"`
	Cmd []string          `yaml:"cmd" json:"cmd"`
	Dir string            `yaml:"dir,omitempty" json:"dir,omitempty"`
	Env map[string]string `yaml:"env,omitempty" json:"env,omitempty"`
}

type Variant struct {
	ID    string            `yaml:"id" json:"id"`
	Env   map[string]string `yaml:"env,omitempty" json:"env,omitempty"`
	Setup []string          `yaml:"setup,omitempty" json:"setup,omitempty"`
}

type Basket struct {
	Name     string    `yaml:"basket" json:"basket"`
	Reps     int       `yaml:"reps" json:"reps"`
	Tasks    []Task    `yaml:"tasks" json:"tasks"`
	Variants []Variant `yaml:"variants" json:"variants"`
}

type Cell struct {
	Task    Task
	Variant Variant
	Rep     int
	RunID   string
	Labels  map[string]string
}

func Load(path string) (Basket, string, error)
func (b Basket) Cells() []Cell

type ManifestEntry struct {
	RunID      string    `json:"run_id"`
	Task       string    `json:"task"`
	Variant    string    `json:"variant"`
	Rep        int       `json:"rep"`
	ExitCode   int       `json:"exit_code"`
	SessionID  string    `json:"session_id,omitempty"`
	Marked     bool      `json:"marked"`
	BasketHash string    `json:"basket_hash"`
	FinishedAt time.Time `json:"finished_at"`
	Note       string    `json:"note,omitempty"`
}

type Manifest struct {
	Path string
}

func (m Manifest) Append(e ManifestEntry) error
func (m Manifest) Completed() (map[string]ManifestEntry, error)
```

**Binding semantics:**

- `Load` reads the YAML, validates (non-empty basket name matching label-key-value constraints; reps ≥ 1; ≥1 task and ≥1 variant; IDs non-empty, unique within their list, valid as label VALUES i.e. ≤256 bytes; every task has non-empty `Cmd`), and returns the basket plus the **basket hash** = hex sha256 of the raw file bytes. Validation errors name the offending field.
- `Cells()` expands in file order: for each task, for each variant, for rep 1..Reps. `RunID` = `bench-<basket>-<task>-<variant>-r<rep>` (deterministic — resume depends on it). `Labels` = `{"basket": name, "task": task.ID, "variant": variant.ID, "rep": strconv.Itoa(rep)}`.
- `Manifest.Append` opens O_APPEND|O_CREATE, writes one JSON line, closes (crash-safe incremental). `Completed()` reads the file if it exists (absent → empty map), keyed by RunID, last entry wins; a malformed line is an error (corrupt manifest must not silently resume).

### Task 2: `catacomb bench` command — cell execution with markers

**Files:**

- Create: `cmd/catacomb/bench.go`, `cmd/catacomb/bench_test.go`
- Modify: `cmd/catacomb/streamjson.go` (runChild gains a session-id observer hook — smallest possible change), `cmd/catacomb/root.go` (wire under Advanced)

**Contract:**

- `catacomb bench <basket.yaml> [--manifest <path>] [--resume] [--fail-fast] [--dry-run]` — default manifest path `<basket path>.manifest.jsonl`.
- **runChild observer:** extend `runChild` with a nil-able `observe func(line []byte)` parameter (or a small options struct if cleaner — keep every existing call site compiling with minimal churn) invoked per teed stream-json line; bench uses it with the existing `streamSessionID`-style parsing (that helper lives daemon-side — reimplement the 5-line JSON peek in bench.go or export; prefer a tiny local unexported func in cmd) to capture the FIRST session_id.
- **Cell execution loop** (sequential, in `Cells()` order): skip cells whose RunID is in `Completed()` when `--resume` (a skipped cell prints one line); for each executed cell: run variant `Setup` commands first (plain `exec`, inherit env; setup failure → cell recorded with Note "setup failed", ExitCode from setup, continue or `--fail-fast`); build env = os.Environ + task.Env + variant.Env (variant wins) + `CATACOMB_LABELS=<canonical merged>` (use `model.FormatLabels` on the cell labels merged over any ambient labels, flag-style: cell wins) + `CATACOMB_RUN_ID=<cell run id>`; execute `cell.Task.Cmd` via the observer-enabled runChild in `Dir`; after the child's first observed session_id, POST `task:<task.ID>` `start` marker; after child exit, POST `end` marker (both via the existing `runMark` in-process with the discovery path — markers are best-effort: mark failure or no session id observed → `Marked:false` + Note, never a cell failure); append ManifestEntry.
- `--dry-run` prints the expansion (RUN_ID/TASK/VARIANT/REP table) and exits 0 without executing.
- **Epilogue** (after all cells, not on `--dry-run`): print two ready-made commands — `catacomb baseline set <basket>-<variantA> --label basket=...,variant=<variantA>` for the FIRST variant and `catacomb regress --baseline label:basket=...,variant=<firstVariant> --candidate label:basket=...,variant=<secondVariant>` when ≥2 variants (single-variant baskets get only the baseline hint).
- Exit codes per Global Constraints. Operational errors: unreadable basket, manifest IO, `--resume` with a manifest whose BasketHash ≠ current basket hash (changed basket invalidates resume — explicit error telling the user to delete the manifest or revert).
- Tests (cobra-level, using the `execCommand` seam + a fake mark transport): expansion→execution order; labels/run-id env reach the child (helper-process echo pattern from streamjson_test.go); resume skips; hash-mismatch resume error; fail-fast stops; setup failure recorded + continues; marker posted on observed session id (fake daemon HTTP server records /v1/mark bodies) and Marked:false without session id; dry-run table; epilogue content for 1 and 2 variants.

HARD note: `runMark` reads discovery via `clientDiscoveryPath()` — check how mark tests fake the daemon (grep mark_test in cmd/catacomb if present, else daemon tests) and reuse the pattern; if `runMark`'s coupling is awkward, extract a thin `postMark(discoveryPath string, sessionID, name, boundary string) error` shared by both commands (behavior-preserving for `mark`).
